1 한눈에 보기

기존 MIS 시스템(mis.greenoilinc.com)의 데이터를 활용해, 서버 코드 변경을 최소화하면서 실시간 대시보드·분석·자동 알림을 구축했습니다. 핵심 설계 원칙:

  • 기존 인프라 무침해 — MIS 서버엔 읽기 전용 API(orders.php) 한 곳만 확장
  • 정적 호스팅 + 서버리스 — 별도 서버 운영 비용 0 (GitHub Pages + Vercel 무료 티어)
  • API 키 노출 방지 — 키는 Vercel 서버리스 함수의 환경변수에만 존재, 브라우저엔 절대 노출 안 됨

6 핵심 자동화 로직

🔒 8시 Forecast Lock

타겟(forecast)은 운영 중 수시로 바뀌므로, 매일 오전 8시 값을 스냅샷으로 고정합니다. 이후 하루 종일 동일 타겟 대비 달성률을 측정.

  • 저장: tbl_forecast_cache (날짜별)
  • 드라이버별 상한·보정 규칙 적용
  • 과거 날짜는 확정 히스토리 사용

💬 Google Chat 자동 알림

매일 오전 8시(월~토), 각 드라이버의 Google Chat Space로 오늘의 목표량을 자동 발송합니다.

  • Vercel Cron 0 12 * * 1-6
  • 드라이버별 Webhook URL
  • 스냅샷과 동일 값 사용 → 일관성 보장
